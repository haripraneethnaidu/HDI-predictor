# Human Development Index (HDI) Predictor

A Flask + scikit-learn web app that predicts a country's HDI score
(and category) from Life Expectancy, Mean Years of Schooling,
Expected Years of Schooling, and GNI Per Capita.

```
hdi_predictor/
├── requirements.txt
├── data/
│   └── hdi_dataset.csv          <- generated by 01_data_collection.py
├── 01_data_collection.py        <- STEP 2
├── 02_eda.py                    <- STEP 3
├── 03_preprocess_train.py       <- STEPS 4, 5, 6
├── model/
│   └── HDI.pkl                  <- trained model, pickle-serialized
├── static/
│   ├── css/style.css
│   └── plots/                   <- EDA + evaluation charts (.png)
├── templates/
│   ├── indexnew.html            <- prediction form / home page
│   └── resultnew.html           <- result display page
└── app.py                       <- STEP 7, Flask application
```

## Step 1 — Environment Setup & Package Installation

```bash
python3 -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate

pip install -r requirements.txt
# installs: numpy, pandas, matplotlib, seaborn, scikit-learn, Flask
```

## Step 2 — Dataset Collection & Understanding

```bash
python 01_data_collection.py
```

This creates `data/hdi_dataset.csv` with columns:
`Country, Life_Expectancy, Mean_Years_Schooling, Expected_Years_Schooling,
GNI_Per_Capita, HDI_Score, HDI_Category`.

> To use the real Kaggle dataset instead of the bundled representative one:
> download a CSV such as
> [Human Development Index Dataset](https://www.kaggle.com/datasets/iamsouravbanerjee/human-development-index-dataset),
> rename its columns to match the schema above, and overwrite
> `data/hdi_dataset.csv`. Steps 3–6 will then run on the real data with no
> other changes.

## Step 3 — Data Visualization & Analysis (EDA)

```bash
python 02_eda.py
```

Generates into `static/plots/`:
- `distplots.png` — distribution of each numeric feature
- `stripplots.png` — feature spread by HDI category
- `heatmap.png` — correlation matrix
- `scatterplots.png` — each feature vs. HDI score
- `category_counts.png` — count of countries per HDI category

## Step 4 & 5 & 6 — Preprocessing, Model Training, Saving

```bash
python 03_preprocess_train.py
```

This script:
1. Fills missing numeric values with the column **mean**
2. Label-encodes `HDI_Category` (`Low < Medium < High < Very High`)
3. Selects the 4 independent variables + `HDI_Score` as the target
4. Splits the data **75/25** train/test
5. Trains a **Linear Regression** model
6. Prints **R-squared**, MAE, RMSE for train/test, and the actual-vs-predicted plot
7. Serializes the trained model (plus the label encoder and metadata) to
   `model/HDI.pkl` with `pickle`, so the Flask app can load it instantly at
   runtime without retraining.

## Step 7 — Run the Flask Web Application

```bash
python app.py
```

Then open **http://127.0.0.1:5000**

- `GET /` — home page with the prediction form; pick a country from the
  dropdown to auto-fill its stats, or enter your own numbers
- `POST /predict` — validates input, runs the pickled model, renders
  `resultnew.html` with the predicted HDI score and category
- `GET /country/<name>` — small JSON helper used by the dropdown's
  auto-fill behavior

## Retraining after replacing the dataset

If you swap in real Kaggle data, just re-run steps 2–4 in order:

```bash
python 01_data_collection.py   # skip this if you dropped in your own CSV
python 02_eda.py
python 03_preprocess_train.py
python app.py
```
